Boolean-based SQL Injection, SQL Injection saldırılarında mantıksal değerlendirmeler üzerinden bilgi edinmeye çalışan, veritabanı yapısındaki güvenlik açıklarından faydalanan etkili bir tekniktir. Bu teknik, hedef sistemdeki veri tabanının hassas içeriklerini açığa çıkarmak için sunucudan gelen mantıksal yanıtları (True/False) yanıtları kullanır.

Temel Adımlar

**1. Giriş Noktasını Belirleme:** Bu tür bir saldırı, doğrudan kullanıcı girdisi alan sorgulara yönelik yapılır. Öncelikle, hedef alınacak giriş noktası belirlenir; kullanıcı adı veya arama kutusu gibi alanlar incelenir.

**2. Mantıksal Yanıt Değerlendirmesi:** Saldırgan, sorguyu manipüle ederek yanıtın mantıksal sonuçlarına göre veri tabanının yapısı ve içeriği hakkında bilgi edinmeye çalışır. Örneğin, sorguya eklenen koşul doğru olduğunda yanıt farklılaşırsa, saldırgan önemli verileri ele geçirebilir.

**3. Mantıksal Operatörlerle Manipülasyon:** Bu tür saldırılarda sorgular, mantıksal operatörler (AND, OR) ve True/False koşulları kullanılarak manipüle edilir. Örneğin:

- **Koşullu Sorgu (Doğru Koşul):**

```auto
SELECT * FROM products WHERE id = 1 AND 1=1;
```

Bu sorgu her zaman True döneceği için veri tabanından sonuç döner.

- **Koşullu Sorgu (Yanlış Koşul):**

```auto
SELECT * FROM products WHERE id = 1 AND 1=2;
```

Saldırgan bu iki tür sorgu arasındaki yanıt farkını gözlemleyerek veri tabanında hassas içerikleri bulabilir.

Uygulama Örneği

Bir uygulamada, kullanıcı girişi için aşağıdaki gibi bir sorgu kullanıldığını varsayalım.

```auto
SELECT * FROM users WHERE username = 'admin' AND password = 'password123';
```

Saldırgan hedef aldığı site ile ilgili hiçbir username, parola bilgisini bilmeden aşağıdaki girdiyi kullanarak sorguyu manipüle eder ve kullanıcı adını öğrenmeye çalışır.

**Saldırgan Girişi**

**username='admin' AND 1=1 --**

**password=anything**

SQL sorgusunun son durumu:

```auto
SELECT * FROM users WHERE username = 'admin' AND 1=1 --' AND password = 'anything';
```

Eğer **admin** olarak bir kullanıcı varsa ve **AND 1=1** koşulu her zaman True döneceğinden, sorgu başarılı olur ve şifre kontrolü atlanmış olur.
